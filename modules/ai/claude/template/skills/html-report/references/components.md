# HTML Report Components

自己完結HTMLレポートを組み立てるためのパーツ集。レポートの内容に合わせて
必要なものだけを選び、`<body>` の中に自由な順序・構成で組み合わせる。
全部使う必要はない。

## Base（毎回そのまま入れる）

CSS変数とタイポグラフィの土台。どのレポートでも `<head>` にこれを入れてから、
使うコンポーネントのCSSだけ追記する。

```html
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<style>
  :root {
    --bg: #ffffff;
    --fg: #1a1a1a;
    --muted: #6b7280;
    --border: #e5e7eb;
    --card-bg: #f5f5f7;
    --code-bg: #f0f0f2;
    --accent: #2563eb;
  }
  @media (prefers-color-scheme: dark) {
    :root {
      --bg: #1a1a1a;
      --fg: #e5e7eb;
      --muted: #9ca3af;
      --border: #333333;
      --card-bg: #242424;
      --code-bg: #1f1f1f;
      --accent: #60a5fa;
    }
  }
  * { box-sizing: border-box; }
  body {
    background: var(--bg);
    color: var(--fg);
    font-family: system-ui, -apple-system, "Segoe UI", sans-serif;
    line-height: 1.6;
    max-width: 900px;
    margin: 0 auto;
    padding: 2rem 1.25rem 4rem;
  }
  h1 { font-size: 1.75rem; margin-bottom: .25rem; }
  h2 { font-size: 1.3rem; margin-top: 2.5rem; padding-bottom: .4rem; border-bottom: 1px solid var(--border); }
  h3 { font-size: 1.05rem; margin-top: 1.5rem; }
  a { color: var(--accent); }
</style>
```

ダーク対応が不要と分かっている場合以外は、必ず両方の配色を書く（OSのテーマに
自動追従させるため）。

## Header / Footer

```html
<header>
  <h1>{{タイトル}}</h1>
  <p style="color: var(--muted); font-size: .9rem;">Generated: {{日時}} / Scope: {{対象範囲}}</p>
</header>

...本文...

<footer style="margin-top: 3rem; padding-top: 1rem; border-top: 1px solid var(--border); color: var(--muted); font-size: .8rem;">
  Generated by html-report skill.
</footer>
```

## Summary cards

3〜5個程度の数値サマリーがあるときだけ使う。指標が無理に作れる場合以外は
省略してよい。

```css
.summary-cards { display: grid; grid-template-columns: repeat(auto-fit, minmax(150px, 1fr)); gap: .75rem; margin: 1.5rem 0; }
.card { background: var(--card-bg); border: 1px solid var(--border); border-radius: 10px; padding: 1rem 1.1rem; }
.card .card-value { font-size: 1.6rem; font-weight: 700; }
.card .card-label { font-size: .8rem; color: var(--muted); margin-top: .15rem; }
```

```html
<div class="summary-cards">
  <div class="card"><div class="card-value">12</div><div class="card-label">Passed</div></div>
  <div class="card"><div class="card-value">2</div><div class="card-label">Failed</div></div>
</div>
```

## Badge

重要度・ステータスなど分類ラベルをテキスト中にインラインで置く。

```css
.badge { display: inline-block; padding: .15rem .6rem; border-radius: 999px; font-size: .75rem; font-weight: 600; white-space: nowrap; }
.badge-high   { background: #fee2e2; color: #991b1b; }
.badge-medium { background: #fef3c7; color: #92400e; }
.badge-low    { background: #dcfce7; color: #166534; }
.badge-info   { background: #dbeafe; color: #1e40af; }
@media (prefers-color-scheme: dark) {
  .badge-high   { background: #4c1d1d; color: #fca5a5; }
  .badge-medium { background: #4a3a0f; color: #fcd34d; }
  .badge-low    { background: #14321f; color: #86efac; }
  .badge-info   { background: #1e3a5f; color: #93c5fd; }
}
```

```html
<span class="badge badge-high">High</span>
```

## Callout

注記・警告・危険を目立たせるボックス。無印は補足情報、`warning`/`danger` は
段階的な注意喚起。

```css
.callout { border-left: 4px solid var(--accent); background: var(--card-bg); padding: .8rem 1rem; border-radius: 6px; margin: 1.25rem 0; }
.callout.warning { border-color: #d97706; background: #fffbeb; }
.callout.danger  { border-color: #dc2626; background: #fef2f2; }
@media (prefers-color-scheme: dark) {
  .callout.warning { background: #332a12; }
  .callout.danger  { background: #3a1c1c; }
}
.callout p:first-child { margin-top: 0; }
.callout p:last-child { margin-bottom: 0; }
```

```html
<div class="callout">補足情報。</div>
<div class="callout warning">注意が必要な内容。</div>
<div class="callout danger">重大な問題。</div>
```

## Table

行数・列幅が読めないため、必ず `overflow-x: auto` のラッパーで包む
（本文全体を横スクロールさせない）。

```css
.table-wrap { overflow-x: auto; margin: 1rem 0; }
table { border-collapse: collapse; width: 100%; font-size: .92rem; }
th, td { border: 1px solid var(--border); padding: .5rem .75rem; text-align: left; }
thead th { background: var(--card-bg); }
```

```html
<div class="table-wrap">
  <table>
    <thead><tr><th>項目</th><th>値</th></tr></thead>
    <tbody><tr><td>A</td><td>1</td></tr></tbody>
  </table>
</div>
```

## Code block

```css
pre { background: var(--code-bg); border: 1px solid var(--border); border-radius: 8px; padding: 1rem; overflow-x: auto; }
code { font-family: ui-monospace, SFMono-Regular, Menlo, Consolas, monospace; font-size: .88rem; }
:not(pre) > code { background: var(--code-bg); padding: .1rem .35rem; border-radius: 4px; }
```

```html
<pre><code>example code block</code></pre>
```

コードの中身に `<`, `>`, `&` が含まれる場合は `&lt;` `&gt;` `&amp;` に
エスケープすること（ログやソース片をそのまま貼るときに崩れないようにするため）。

## Collapsible（折りたたみ）

長いログや補足を畳んでおきたいときに使う。JS不要。

```css
details { border: 1px solid var(--border); border-radius: 8px; padding: .6rem 1rem; margin: 1rem 0; }
details summary { cursor: pointer; font-weight: 600; }
details[open] summary { margin-bottom: .5rem; }
```

```html
<details>
  <summary>詳細を表示</summary>
  <p>折りたたみ内のコンテンツ。</p>
</details>
```

## Table of contents（長いレポート向け）

セクション数が多い（目安5個以上）場合のみ、先頭に置く。

```html
<nav style="margin: 1.5rem 0;">
  <strong>目次</strong>
  <ul>
    <li><a href="#section-1">セクション1</a></li>
    <li><a href="#section-2">セクション2</a></li>
  </ul>
</nav>

<section>
  <h2 id="section-1">セクション1</h2>
  ...
</section>
```
